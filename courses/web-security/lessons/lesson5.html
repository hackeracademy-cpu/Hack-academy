CSRF Protection

What is CSRF?

Cross-Site Request Forgery (CSRF) is an attack that forces authenticated users to submit unwanted requests to a web application. These attacks exploit the trust that a web application has in an authenticated user's browser.

How CSRF Works

Common CSRF Attack Vectors:

  • Hidden forms in malicious websites
  • Automatic form submissions
  • Cross-domain image requests
  • Malicious links in emails

const app = express();

// Setup CSRF protection

// Add CSRF token to form

// Validate CSRF token on submission
app.post('/submit', (req, res) => {

    res.send('Form submitted successfully');

Best Practices

Key Protection Measures:

  • Use CSRF tokens in forms
  • Implement SameSite cookie attribute
  • Verify Origin and Referrer headers
  • Use custom request headers for AJAX
