Understanding HTTP

HTTP Protocol Basics

HTTP (Hypertext Transfer Protocol) is the foundation of data communication on the web. Understanding how it works is crucial for web security.

Key HTTP Components:

  • Request Methods (GET, POST, PUT, DELETE)
  • Headers
  • Status Codes
  • Request/Response Body

HTTP Headers and Security


# Security-related HTTP headers
Content-Security-Policy: default-src 'self'
X-Frame-Options: DENY
X-XSS-Protection: 1; mode=block
X-Content-Type-Options: nosniff
Strict-Transport-Security: max-age=31536000; includeSubDomains

These headers help protect against various attacks like clickjacking, XSS, and MIME-type confusion.

HTTPS and SSL/TLS

HTTPS adds encryption to HTTP using SSL/TLS protocols. This ensures:

  • Data confidentiality
  • Data integrity
  • Server authentication

// Node.js HTTPS server example
const https = require('https');
const fs = require('fs');

const options = {
    key: fs.readFileSync('private-key.pem'),
    cert: fs.readFileSync('certificate.pem')
};

https.createServer(options, (req, res) => {
    res.writeHead(200);
    res.end('Hello Secure World\n');
}).listen(443);

Common HTTP Vulnerabilities

Security Risks:

  • Man-in-the-Middle Attacks
  • Session Hijacking
  • HTTP Response Splitting
  • Request Smuggling
